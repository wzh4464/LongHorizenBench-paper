\section{Introduction}
\label{sec:introduction}

Coding agents have moved quickly from interactive code completion to autonomous repository editing. Public leaderboards now report strong results on bug-repair tasks, and engineering organizations increasingly ask whether the same tools can take over larger units of software work. The open question is not whether an agent can suggest a local patch, but whether it can produce a candidate pull request (PR) for an industrial, system-level feature that a senior reviewer would merge with at most light revision.

\paragraph{Industrial, system-level feature implementation.}
In this paper, an \emph{industrial, system-level feature task} is a merged PR that implements new behavior in a production repository, usually requiring multi-file coordination, repository-specific conventions, tests, and architecture-aware integration. This differs from localized bug fixing: the agent must discover the relevant files, preserve cross-module invariants, and stop only when the feature is complete enough for review.

\paragraph{Why existing benchmarks are insufficient.}
Current coding-agent benchmarks emphasize issue resolution and small, well-localized repairs. SWE-bench Verified~\cite{jimenez2024swebench,chowdhury2024swebenchverified} supplies real repository issues, but many tasks target a small file set and reward minimal local edits. Feature-level and long-horizon benchmarks such as SWE-bench Pro~\cite{swebenchpro2025}, SWE-EVO~\cite{sweevo}, and FeatBench~\cite{featbench2025} move closer to feature work, yet the merge-readiness of commercial agents on industrial-scale, system-level PRs remains underexplored. We therefore evaluate agents on tasks whose median main-corpus change touches 57 ground-truth (GT) files and 6{,}905 lines of diff.

\paragraph{Benchmark and evaluation.}
We construct a two-tier benchmark with 62 tasks. The 50-task main corpus, LongBench, contains long-horizon feature PRs from 15 production-grade open-source projects. A 12-task industrial calibration subset covers accelerator-operator, distributed-training, and platform-engineering tasks and is used to validate the scoring rubric against blinded senior reviewers. We evaluate four agents---Claude Code (Opus 4.6), OpenAI Codex (GPT-5.4), Cursor Composer-2, and OpenCode/GLM-5.1---under short and long prompt variants, producing 496 runs. Each run receives a deterministic handwritten ground-truth (HW/GT) coverage report, then four independent LLM evaluators score functional correctness, completeness/coverage, and behavioral equivalence using the report and the diffs; a conservative majority rule assigns SE-PASS, PARTIAL, or FAIL.

\paragraph{Key findings overview.}
\begin{itemize}
  \item \textbf{Merge-ready feature PRs are rare.} The main corpus yields 4/400 SE-PASS outcomes (1.0\%), all on one trivially scoped single-file task. Excluding that task, the rate is 0/392.
  \item \textbf{Task shape dominates product choice.} On the main corpus all four agents tie at 1.0\% SE-PASS. The 12-task calibration subset shows that subsystem-contained operator/framework tasks occasionally pass, while Kubernetes-class platform tasks return 0/32 SE-PASS.
  \item \textbf{Long prompts improve drafts, not hard-task pass rates.} Detailed prompts raise mean scores by +0.99 to +2.12 on the main corpus, but produce no SE-PASS on non-trivial long-horizon tasks.
  \item \textbf{Evaluation requires calibration.} Deterministic HW/GT coverage reports ground the completeness score, while pairwise LLM-judge agreement still ranges from 42.5\% to 74.4\%; the four-judge panel reaches 92.7\% SE-PASS-vs-non-SE-PASS agreement with blinded senior reviewers on the calibration subset.
\end{itemize}

\paragraph{Contributions.}
This paper makes three contributions:
\begin{enumerate}\itemsep0pt
  \item A two-tier benchmark and 496-run empirical evaluation for industrial, system-level feature implementation, combining a reproducible 50-task main corpus with a 12-task human-calibrated industrial subset.
  \item A hybrid evaluation protocol that combines deterministic HW/GT coverage reports, three 0--5 scoring dimensions, four LLM judges, conservative tie-breaking, anti-leakage controls, and blinded human review.
  \item Empirical findings and a failure analysis suggesting that current agents are useful mainly as subsystem-contained task drafters, not autonomous feature owners, and require structured specifications, scope/test checks, and calibrated multi-evaluator review.
\end{enumerate}
