\section{Conclusion}
\label{sec:conclusion}

Under our benchmark of 12 industrial feature-implementation tasks
spanning four languages and three complexity tiers, no agent
configuration achieves reliable autonomous feature implementation.
The best configurations (Claude Opus~4.6 and Codex GPT-5.4) each reach 12.5\% PASS
under our rubric; the overall rate across 96 experiments is 8.3\%.
Task complexity exposes an absolute capability cliff: top agents pass
25.0\% of low-complexity tasks but 0\% of high-complexity ones.
Root-cause analysis shows that 84.1\% of non-PASS outcomes stem from
engineering-addressable gaps---primarily incomplete file coverage---rather
than fundamental model-capability limitations.

These findings carry a concrete industrial consequence.
This research informed our team's decision to adopt
\emph{agent-assisted} rather than agent-autonomous workflows, with
mandatory human review gates for API design, test coverage, and
feature lifecycle decisions.
At the same time, the harness engineering results are encouraging:
detailed input specifications improve pass rates by up to 25.0
percentage points for capable agents, confirming prompt granularity
as the single strongest performance lever.

Future work should pursue three directions:
(1)~\emph{stronger harness}---fix-cycle loops that feed validation
failures back as repair instructions, enforced test-generation
mandates, and language-aware output validation;
(2)~\emph{broader evaluation}---a larger and more diverse task set
covering additional languages, repository types, and complexity
ranges to test generalizability;
(3)~\emph{human--agent collaboration modes}---measuring productivity
and quality when agents operate under human supervision rather than
autonomously, which better reflects the workflow our industrial
experience suggests is practical today.
