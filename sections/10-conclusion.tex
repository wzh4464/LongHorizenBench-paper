\section{Conclusion}
\label{sec:conclusion}

The primary outcome of this work is a four-gate deployment policy now in production at Huawei's Ascend NPU stack: a task-class gate that routes only bounded operator-tier work to agents, a mandatory long-form specification template, a pre-review CI gate enforcing file scope and test coverage, and a 3-of-4 LLM-judge acceptance panel. Each gate is justified by a specific empirical finding from a 496-run industrial audit, and the pipeline as a whole converts agents from unreliable autonomous coders into useful \emph{gated drafters} for bounded tasks.

The audit evidence behind the policy is clear: on the 50-task main corpus of long-horizon feature tasks, only 1.0\% of runs reach SE-PASS, and excluding one trivially scoped single-file outlier the rate is 0/392 = 0.0\%. The failure mass splits into semantic errors (44.7\%) and coverage gaps (25.5\%) that the scope/test gate directly targets. The 12-task calibration subset confirms rubric discriminant validity (bounded operator tasks pass, Kubernetes-class platform work returns 0/32). Whether measured by strict reference equivalence or by senior-reviewer acceptance, current agents are not ready for autonomous feature delivery---but the routing pipeline makes their output reviewable.

\paragraph{Future work.}
The audit leaves three concrete follow-ups, each addressing one of the threats catalogued in \S\ref{sec:threats}.
(i) \emph{Reproducibility:} re-execute each SE-PASS-yielding cell $k=5$ times to convert the headline point estimate into a confidence interval.
(ii) \emph{Construct validity:} run a randomised human re-rating on the 146 main-corpus PARTIAL outputs to quantify the gap between SE-PASS and a relaxed reviewer-acceptable verdict (RA-PASS).
(iii) \emph{Calibration extension:} repeat the 96-cell human-rater pilot on a CapBench stratified sample so the panel-vs-human alignment can be reported across the main corpus rather than only the calibration subset.
Each closes a known gap that limits how strongly the deployment-policy claims in \S\ref{sec:industrial-deployment} generalise outside Huawei's repositories.
