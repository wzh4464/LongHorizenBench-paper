\section{Threats to Validity}\label{sec:threats}

\textbf{Single-run stochasticity.} Each task--agent--prompt cell was run once; aggregate patterns (Kubernetes 0/32, long-prompt PASS lift, harness-tier ordering) are stable across all 62 tasks, but individual cell verdicts may shift on re-run. The deployment-relevant signal (PASS rate of $\sim$1--2\%) sits well below typical re-run variance for these agents on bug-fix benchmarks.

\textbf{Product-level confound.} Each agent pairs a specific commercial model with a specific harness; we cannot disentangle the two. We therefore phrase observations such as ``harness leverage is substantial'' as observations on \emph{deployed products}, not as causal claims about harness-vs-model contribution. The 92.7\% PASS-vs-non-PASS agreement between our judge panel and three blinded senior reviewers indicates that the verdicts themselves are not a panel artefact.

\textbf{Strict production-equivalence rubric.} Our rubric counts a run as PASS only if it implements the reference behaviour and updates the same handwritten files (within the structural-equivalence rules in \S\ref{sec:study-design}). A valid alternative implementation that diverges from the upstream PR will be marked non-PASS even if a human reviewer would accept it. The 1.6\% headline therefore lower-bounds merge-readiness rather than measures it directly.
