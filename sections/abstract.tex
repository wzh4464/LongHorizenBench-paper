Industrial teams now ask a deployment question, not a benchmark question: \emph{can current coding agents produce feature pull requests}---multi-file, API-changing, test-bearing changes---\emph{that engineers would actually merge}? We answer this for Huawei's coding-agent rollout by auditing four commercial agents on 62 real feature-implementation tasks and reporting where they break.
We treat each agent as a \emph{harness} on top of a frontier model: Claude Code (Opus~4.6), Codex CLI (GPT-5.4), Cursor (Composer-2), and OpenCode (GLM-5.1).
We use \emph{harness} to mean the system around the model---planning, retrieval, tool use, edit policy, validation hooks, and review gates.
The benchmark covers 62 tasks (12 from Huawei CANN, MindSpeed, torch\_npu, and Kubernetes workflows; 50 from our open-source extension {CapBench}, spanning Kafka, CPython, Airflow, TensorFlow, Django, and others) and a controlled long/short prompt comparison, yielding $4\!\times\!62\!\times\!2=496$ runs.
Each run is scored by a four-agent LLM panel using a three-dimension production-equivalence rubric (functional correctness, completeness, behavioral equivalence) and a strict majority verdict.
Across 496 runs, only 8 reach a production-equivalent verdict (1.6\%); the strongest harnesses (Claude Code, Codex CLI) pass 2.4\% each, the weaker harnesses (Cursor, OpenCode) pass under 1\%.
Every PASS run concentrates on four of the 62 tasks, and the entire Kubernetes track yields 0/32 PASS.
The dominant failure mode is harness-side, not model-side: agents typically infer the correct goal from the task description but cover only 30--60\% of the ground-truth file set before stopping, omit regression tests and generated artifacts, and skip cross-module wiring.
Long prompts lift the strongest harnesses from 0\% to 4.8\% PASS while leaving Cursor and OpenCode flat, suggesting that gains from richer specification are mediated by the harness, not the underlying model.
We close with a four-step harness recipe (long-form task spec, scope contracts, output-side test/coverage gating, multi-evaluator acceptance) that follows directly from where today's agents fail in our audit.
The takeaway for industry teams is concrete: ship coding agents now for low-coupling tasks, but invest harness engineering---not just larger models---to close the feature-PR gap.
