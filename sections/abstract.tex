Coding agents now achieve strong results on public bug-fix benchmarks, but it remains unclear whether they can produce merge-ready pull requests for industrial, system-level feature work. We study this question with a 62-task benchmark that combines a 50-task main corpus of long-horizon feature pull requests from 15 open-source projects with a 12-task industrial calibration subset from accelerator, distributed-training, and platform-engineering codebases.

We evaluate four commercial coding agents under two prompt granularities, yielding 496 end-to-end runs. Each run is evaluated with a deterministic handwritten ground-truth (HW/GT) coverage report and a four-LLM evaluator panel that scores functional correctness, completeness/coverage, and behavioral equivalence to the merged reference implementation; the panel is calibrated against three blinded senior reviewers on all 96 calibration-subset runs, reaching 92.7\% SE-PASS-vs-non-SE-PASS agreement. The main corpus yields only 4/400 SE-PASS outcomes (1.0\%), all on one trivially scoped single-file task; excluding that outlier, no agent produces a merge-ready candidate on the remaining 392 main-corpus runs. The calibration subset confirms the task-shape effect: bounded operator/framework tasks occasionally pass, while Kubernetes-class platform tasks return 0/32 SE-PASS.

The empirical results suggest that current coding agents are not ready for autonomous industrial feature delivery. They are more plausibly useful as bounded-task drafters when paired with structured specifications, scope/test checks, and calibrated multi-evaluator review.
