LLM-based coding agents now resolve over 70\% of SWE-bench bug-fix tasks, prompting industry interest in deploying them for larger-scale feature implementation.
Yet no rigorous evaluation exists for multi-file feature development in industrial codebases.
We present an empirical assessment of four agent configurations---Claude Opus~4.6, GPT-5.4 (Codex), Cursor Composer-2, and GLM-5.1 (OpenCode)---across 12 feature-implementation tasks drawn from four production repositories (Huawei CANN, MindSpeed, torch\_npu, and Kubernetes) in C++, Python, Go, and AscendC, yielding 96 experiments each independently scored by three LLM evaluators for cross-validation.
The best configurations achieve 12.5\% PASS under our rubric, and overall only 8.3\% of experiments produce production-equivalent implementations.
Task complexity exposes an absolute capability cliff: top agents pass 25.0\% of low-complexity tasks but 0\% of high-complexity ones.
Root-cause analysis reveals that 84.1\% of failures stem from engineering-addressable gaps---primarily incomplete file coverage---rather than fundamental model-capability limitations.
Detailed prompts improve pass rates by up to 25.0 percentage points for capable models, confirming prompt granularity as the single strongest performance lever.
Our findings indicate that coding agents are not ready for autonomous deployment on feature implementation, but that structured specifications and process-side interventions offer a practical direction to bridge the gap.
