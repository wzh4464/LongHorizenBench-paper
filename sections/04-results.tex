\section{Empirical Results of Agent Capability}
\label{sec:results}

This section answers RQ1 and RQ2. We first measure whether current agents produce merge-ready feature PRs, then analyze how task shape affects the result.

\subsection{Effectiveness of Existing Agents (RQ1)}
\label{sec:results-main}

The main corpus contains 400 runs (50 tasks $\times$ 4 agents $\times$ 2 prompts). Table~\ref{tab:main-agent} shows that all four agents reach the same main-corpus SE-PASS count: 1/100 each. The only passing main-corpus task is T10, a single-file Presto benchmark addition; excluding T10, no agent reaches SE-PASS across the remaining 392 runs.

The gap between mean score and SE-PASS is important. Claude and Codex produce more PARTIAL outputs than OpenCode, and their average scores are meaningfully higher. In a drafting workflow, this matters: a partial implementation can reduce exploration cost for a human engineer. In an autonomous feature-delivery workflow, however, the same outputs remain insufficient because one missing subsystem, test layer, or API registration can invalidate the PR. We therefore report mean score as a secondary draft-quality indicator and SE-PASS as the primary merge-readiness indicator.

The single passing main-corpus task also acts as a protocol sanity check. T10 is not an industrial-scale feature; it is a single-file benchmark addition. The fact that all agents pass it under the long prompt shows that the evaluation is not mechanically impossible. The failure is concentrated in the intended target regime: tasks requiring sustained cross-file reasoning and closure.

Tables~\ref{tab:result-diagnostics-family} and~\ref{tab:result-diagnostics-score} provide two diagnostics for interpreting Tables~\ref{tab:main-agent} and~\ref{tab:config-pass}. The family-level table shows that pass rates are shaped more by task family than by product choice: SE-PASS outcomes concentrate in bounded CANN/MindSpeed tasks and the single LongBench outlier, while Kubernetes remains at 0\%. The score-bucket table shows that most runs occupy the low-to-mid score buckets, explaining why mean-score gains often indicate better drafts rather than merge-ready PRs.

\finding{1}{Current commercial coding agents rarely produce merge-ready feature PRs under a strict-equivalence criterion. The main corpus yields 4/400 SE-PASS outcomes (1.0\%), and all four disappear when the trivially scoped T10 task is excluded.}

\subsection{Effect of Task Shape and System-Level Coordination (RQ2)}
\label{sec:results-family}

Task family explains more of the SE-PASS distribution than product choice. Table~\ref{tab:family} separates the 12-task calibration subset from the 50-task main corpus. Bounded accelerator and framework tasks produce a small number of passes, whereas Kubernetes-family platform tasks produce none across 32 runs.

Figure~\ref{fig:capability-cliff} visualizes the same pattern at task level. T10 is the lone main-corpus task above the approximate SE-PASS threshold. The calibration subset shows that low-coordination tasks can occasionally pass, but cross-module platform work remains below the threshold even when the GT file count is lower than some main-corpus tasks. This indicates that task difficulty is not captured by file count alone; cross-component coordination and test scaffolding matter.

The Kubernetes-family tasks illustrate the coordination effect most clearly. They do not simply require many files; they require changes that remain consistent across API behavior, validation, generated surfaces, and tests. Agents often discover part of the relevant area but fail to complete the integration path. Conversely, the passing C and M tasks are bounded enough that a correct local plan can close the feature. This supports a routing interpretation: task family and coordination structure should determine whether an agent is used, not only the apparent size of the ticket.

\finding{2}{SE-PASS outcomes concentrate in bounded tasks. The 8 passing runs cluster in four tasks (C1, C4, M1, and T10), while Kubernetes-class platform tasks return 0/32 and the 49 non-trivially scoped main-corpus tasks return 0/392.}
