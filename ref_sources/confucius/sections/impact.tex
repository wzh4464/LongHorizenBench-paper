\section*{Impact Statement}

This paper advances the field of machine learning by improving the reliability, extensibility, and controllability of LLM-based coding agents through better \emph{agent scaffolding} (orchestration, memory, and tool-use abstractions). 
While these improvements can increase developer productivity and reduce repetitive engineering work, they may also amplify risks common to software automation: generating insecure code, enabling misuse (e.g., accelerating vulnerability discovery or malware development), and causing unintended side effects when interacting with external systems. We mitigate these risks by emphasizing constrained tool interfaces, structured parsing, bounded execution, and auditable traces that support oversight and debugging, but we acknowledge that responsible deployment still requires access control, sandboxing, and human review for high-stakes changes.
